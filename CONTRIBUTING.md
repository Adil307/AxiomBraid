# Contributing

1. Open an issue describing the problem or proposed feature.
2. Create a focused branch and include tests for behavioral changes.
3. Run `python -m pytest` and `python -m ruff check src tests`.
4. Update documentation and `CHANGELOG.md` when the public behavior changes.
5. Avoid committing datasets containing personal, confidential, or proprietary data.

Public API changes require an issue, migration notes, and semantic-version review.
